## 26.4.17

### Changed

- 修复 Windows 自动更新启动安装包时因安装器需要管理员权限而触发 `spawn EACCES` 的问题，改为通过系统提权流程启动安装器
- 调整 Windows 安装包提权清单，避免旧版更新器在创建新安装包进程时被系统拒绝，同时保留现有全用户安装的静默升级提权流程
- 限制内核运行模式过快切换，并串行化主进程内核重启流程，避免直接运行和系统服务来回点击时触发核心控制器未初始化错误
- 显式指定 Windows 可执行文件图标，避免本地无签名打包流程生成默认 Electron 图标

## 26.4.16

### Changed

- 修复 Windows 服务模式启动内核后 `resources\sidecar` 被服务加固为普通用户不可访问，导致直接运行或系统服务模式报 `spawn ENOENT/EPERM` 的问题
- 启动内核和管理服务前会检测并修复已损坏的内置内核目录权限，恢复安装目录下 `mihomo.exe` 与 `mihomo-alpha.exe` 的可读执行权限
- 修正旧服务注册项迁移逻辑，发现服务仍指向开发目录或旧启动参数时会自动更新到安装目录的 `perzike-service.exe`
- 移除 Windows 下复制内核到用户数据目录的 fallback，内置内核路径重新与 Sparkle 保持一致，固定使用安装目录的 `resources\sidecar`

## 26.4.15

### Changed

- 修复应用启动时已处于系统服务模式但旧服务注册项未修正，导致服务不可达后回退本地启动并报 `spawn EPERM` 的问题
- 在服务模式启动内核前先修正并启动 Perzike 服务，确保服务管道可用后再由服务接管内核启动

## 26.4.14

### Changed

- 修复 Windows 服务注册参数顺序，确保安装版服务启动后监听 Perzike 管道而不是 Sparkle 默认管道
- 重新检测已有服务注册命令，发现旧的 `--listen ... service run` 顺序时会自动改写为 `service run --listen ...`

## 26.4.13

### Changed

- 合并内核服务模式支持，补充服务鉴权、安全存储、启动链路、服务事件流和日志采集能力
- 修复 Windows 服务初始化和启动时沿用 Sparkle 默认管道导致 Perzike 无法连接服务的问题
- 补齐内核设置页的运行模式、提权状态、服务状态、内核进程优先级和日志设置，并保持已移除的系统内核选项不恢复
- 统一设置页、系统代理、外部资源和日志页面中相关控件的主题色样式
